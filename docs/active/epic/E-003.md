## Intent — the bigger-picture play

The game rules exist as pure logic in lib/, but nothing is playable yet — there is no screen that reacts and no clock that advances. This epic wires the render+loop+input seam: React Board/Cell components that paint board state to a DOM/CSS grid, a requestAnimationFrame tick that drives gravity, and keyboard handlers for move / rotate / soft-drop / hard-drop. It closes the loop from key-press → core state transition → repaint, turning the assembled engine into something a person can actually hold and play. This is the single highest-leverage move toward a demo right now: P1 says a working game loop outranks everything, and without this seam the core is invisible.

## Value to the design

Realizes end-to-end playability (P1): the first build where the screen responds to input and pieces fall on their own, so the core Tetris rules become a game a stranger can complete unaided — spawn, move, rotate, drop, lock, clear, game-over — all observable on screen.

## Done looks like

Loading the app renders the live board grid; pieces fall automatically under a rAF-driven gravity tick; the keyboard moves, rotates, soft-drops and hard-drops the active piece; locked cells and cleared lines update on screen; and a stranger can play one uninterrupted game from spawn to game-over without touching code or console.

## Context & constraints

Grounds against the current repo: game rules live pure and framework-free in lib/ (per CLAUDE.md source layout and the E-002 core-engine work — [[e-002-decomposition]]); this epic consumes that core and must not reimplement rules in components. Scope is strictly the render + loop + input seam. PREREQUISITE (not hidden scope): a usable pure game-core API in lib/ (board state, spawn/move/rotate/drop/lock/clear/game-over transitions) must exist for this seam to bind to — if it is incomplete, that gap is its own epic, not absorbed here. OUT OF SCOPE / separate epics: the neon/glass visual wow, 60fps animation and line-clear juice (P2); scoring, levels, next-piece preview, hold, ghost piece, high-score persistence (P4); and Vercel deploy (P3). NON-GOALS respected (N-charter): no backend, no accounts, no multiplayer, no native app — this stays a single-player, client-only web seam. Right-sized as ~one 2-hour feature block: three tightly-coupled parts (components, tick, input) over one clean file boundary (components/ + app wiring) that DecomposeEpic can split into parallel tracks.
