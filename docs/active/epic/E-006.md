## Intent — the bigger-picture play

Move the build/runtime and deploy path off the Next.js compiler + Vercel wiring and onto vinext (Cloudflare's Vite-based, Next.js-API-compatible runtime) targeting Cloudflare Workers, so the deploy target matches the charter and the retargeted docs. Install vinext + vite@^7, scaffold the vinext/Vite config (vinext() + @vitejs/plugin-rsc), flip package.json to ESM and point dev/build/start at vinext, run the compatibility check and resolve any App Router findings, retire the Vercel config, and generate + document the Cloudflare Workers deploy path. The game, components, and lib/ logic stay byte-for-byte unchanged — this is a toolchain swap, not a feature or refactor.

## Value to the design

Realizes a working Cloudflare Workers deploy toolchain (P3 shippability): the one deploy path the charter now names becomes real instead of documented-only, unblocking a shareable public URL for the demo while keeping playability and the wow intact (build/lint/vitest all green under the new runtime).

## Done looks like

The app builds, lints, and passes the full vitest suite under vinext (`vinext`-backed dev/build/start); `vinext check` reports no unresolved App Router incompatibilities; `vercel.json` and Next-compiler-specific wiring are gone; a `wrangler.jsonc` is generated and the authenticated `vinext deploy` step is documented as a manual, login-gated follow-up. Observable outcome: a clean production build on the Cloudflare Workers toolchain with no game/logic behavior change.

## Context & constraints

Grounds against the current repo: package.json scripts are still `next dev/build/start` on next@16.2.9 with `vercel.json` present and NO vite.config.ts or wrangler.jsonc — so this migration is unexecuted despite commit 3c13875 having retargeted the docs to vinext/Cloudflare. react is already 19.2.4 and vitest is already the test runner, which lowers risk. Prerequisites (name AS prerequisites, not hidden scope): vinext + vite@^7 install, and vinext's App Router compatibility surface must actually cover this app's usage — if `vinext check` surfaces incompatibilities that need real code changes, that is a signal this is larger than one epic. Non-goals respected: no backend/persistence/auth/analytics added (N1–N4), no gameplay or visual changes, no lib/ logic touched. Explicitly OUT of scope: the final authenticated `vinext deploy` to a live Cloudflare account (requires wrangler/Cloudflare login — a manual human step, documented here but not performed), and any post-deploy URL smoke-testing. Right-sized (PE-7) as ~one 2-hour tooling block; if compatibility findings balloon, split the resolution work into its own follow-up epic rather than inflating this card.
