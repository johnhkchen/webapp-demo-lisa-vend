## Intent — the bigger-picture play

Stand up the Next.js (App Router) + TypeScript foundation the whole demo builds on: a project that boots with `npm run dev`, passes `npm run build` and `npm run lint`, and carries the agreed `app/` · `components/` · `lib/` layout so rendering and pure game logic stay on separable tracks. Lock in the two human-assented technology forks — Tailwind for styling (neon/glass theme to follow) and a DOM/CSS-grid of cell divs as the board renderer — so downstream epics inherit settled ground instead of re-litigating them. This is the keystone: nothing else on the board can be pulled until the skeleton runs.

## Value to the design

Realizes Shippability (P3) at the root — a passing production build and a runnable app are the precondition for a Vercel deploy — and lays the structural track that lets the Playability (P1) game loop and the Wow (P2) visual work proceed in parallel without file-boundary collisions. It converts an empty repo into the substrate every other epic depends on.

## Done looks like

From a clean checkout, `npm run dev` serves the app at localhost:3000, `npm run build` produces a passing production build, and `npm run lint` runs clean; the `app/` · `components/` · `lib/` directories exist with the App Router entry (page/layout/globals); Tailwind is wired and applies; and the page renders a placeholder board as a CSS grid of cell divs. The next signal can be pulled with no scaffolding work remaining.

## Context & constraints

Grounded against the current snapshot: repo has CLAUDE.md/docs but no `src/**` modules, stories, or tickets — this is the first epic (E-001), disjoint from the empty board. Fits one ~2-hour feature block (PE-7). Two forks are already human-assented and must be honored as decisions, not re-opened: Tailwind (over CSS Modules) and DOM/CSS-grid cell divs (over canvas). Prerequisite for essentially all downstream epics; name it as such rather than absorbing their scope. Out of scope (PE-5/PE-6): the actual game loop, tetromino/collision/scoring logic, the neon/glass visual theme, animations, and the Vercel deploy itself — those are separate epics DecomposeEpic will clear. Respects non-goals N1–N4: no backend, accounts, multiplayer, or native shell; the skeleton is a client-only web app. Deliberately does not decompose into stories/tickets — that is DecomposeEpic's output.
