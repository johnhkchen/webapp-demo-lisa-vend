## Intent — the bigger-picture play

Vendor the shared b28-clay kit (`just sync-kit` pulling https://b28.dev/kit/b28-clay.css into the app) and re-plumb RowClear's visual identity through it: map the kit's `--clay-*` tokens into the Tailwind v4 `@theme`, then retone every surface — board as a recessed clay well, next/hold/HUD as raised clay tiles, controls as clay buttons with real press states, Lora/Karla type, warm off-white page — so the game reads as a b28.dev-family app rather than a bespoke neon demo. The seven-piece palette stays its own layer but is retoned to muted hues that sit on cream clay.

## Value to the design

This is the P2 payoff axis directly — right now the 'wow' is a generic dark-neon/glass look with no brand identity; this epic makes the visual language legible as *this maker's* work and family-consistent with other b28.dev properties, which is the whole point of a demo meant to be shown around. It also upgrades P4-adjacent feel: genuine clay press states on controls are exactly the kind of tactile touch that makes a game satisfying to keep playing, not just functional.

## Done looks like

Loading the deployed app shows a warm off-white clay UI end to end — the board is a recessed clay well, the next/hold/HUD panels are raised clay tiles, buttons show a real pressed state on interaction, type is Lora/Karla, and the seven pieces read as muted-but-distinct hues on cream clay — with zero remaining trace of the old dark-neon/glass palette. Running `just sync-kit` followed by a rebuild picks up a kit palette change without any hand-edited component CSS.

## Context & constraints

Grounds against the current theme in app/globals.css: a `:root`/`@theme inline` background-foreground pair, a `@theme static` block of seven oklch neon piece colors (`--color-piece-i..l`) consumed via `bg-piece-*`/`ring-piece-*` utilities in components/Cell.tsx, and a hand-rolled `.glass` panel utility (frosted-blur, translucent fill) used by the scoreboard/preview/hold panels — all of that is the dark-neon/glass material this epic replaces. No `justfile` exists yet in the repo, and there's no Lora/Karla font loading today (globals.css still uses a system-ui stack) — both are prerequisites this epic must stand up, not assume. Treat the b28-clay.css kit URL as an external prerequisite outside this repo's control; the sync step is a build/dev-time fetch (via `just sync-kit`), not a runtime request, so it does not put Cloudflare Workers request-path shippability (P3) at risk, but the kit's availability/shape is a dependency to flag if sync-kit can't be authored against it directly. Purely a CSS/token/component-styling layer — no gameplay-logic, board-state, or `lib/` changes, so it can't regress P1 playability or P5 rigor; respects the charter's non-goals (no backend, no persistence, no scope creep into gameplay). Despite touching many surfaces (board, cell, next/hold/HUD, buttons, piece palette, global tokens), this stays one epic rather than several: each surface lives in its own component file, so DecomposeEpic can slice it into parallel, non-overlapping-file story tracks the way E-007's feel-pack epic did, rather than needing to be split across multiple epics.
