## Intent — the bigger-picture play

Give the start/Ready screen a live self-playing attract-mode: behind the start overlay, a pure typed placement AI auto-plays Tetris at an accelerated drop speed, looping continuously until the player presses Start, at which point it hands off cleanly to a fresh human game. The bot is a pure chooseMove(state): Input[] living in its own lib/ module — it plans the best placement for the current piece using standard board heuristics (aggregate height, holes, bumpiness, completed lines; Dellacherie / El-Tetris style) and drives that plan through the existing typed game core rather than mutating state directly. The attract renderer reuses the existing board rendering; only the driver (auto-stepping the bot's inputs, then relinquishing to human control on Start) is new. This turns a static overlay into a moving showcase and puts a real search/heuristic algorithm on display — the kind of typed, provable-correctness work a minutes-long one-shot structurally won't attempt.

## Value to the design

Advances P2 (the wow): a landing screen that plays a competent game of Tetry on its own is a strong, immediate visual hook that reads as 'finished product' before any interaction. Advances P5 (engineering rigor as proof): a pure, deterministic, unit-testable chooseMove built on branded/typed board state and a real heuristic search is concrete evidence of the many-shot pipeline's reach — it exercises the existing pure core as a driveable API and doubles as benchmark-scorecard substance. Because the bot is pure and framework-free, it lands behind a clean lib/ boundary and is provable in isolation, keeping AI logic separable from the attract renderer and human game loop.

## Done looks like

Loading the deployed game shows, behind the start overlay, the board auto-playing Tetris — pieces are placed by the AI at an accelerated fall speed, lines clear, and the demo loops (resets and continues) without human input; pressing Start cleanly stops the bot and begins a fresh human-controlled game with no residual bot state or input bleed-through. The bot lives as a pure chooseMove in its own lib/ module with unit tests asserting it picks sane placements (fills rows, avoids creating holes/excess height) deterministically for given board+piece inputs. Core human playability, scoring, and the production build stay green. Observable at the epic level: a bystander who opens the URL and never touches the keyboard watches the game play itself.
