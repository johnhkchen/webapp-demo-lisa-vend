## Rationale anchors (from design.md)

- `@vitejs/plugin-react` **must** be `^5.x`, never `^6`: the 6.x peer is `vite ^8`, which
  breaks against `vite@^7`. This single pin is what turns the ERESOLVE into a clean tree.
- `react`/`react-dom` at `^19.2.7` is the minimal bump satisfying the transitive floor
  (`react-server-dom-webpack@19.2.7` peers `react ^19.2.7`).
- All four "hidden" vinext peers (`@mdx-js/rollup`, `@vitejs/plugin-react`,
  `react-server-dom-webpack`, `vite-tsconfig-paths`) are declared explicitly so `npm ci` is
  reproducible from `package.json`, not dependent on npm's auto-peer heuristics.

## `package-lock.json` shape

`package-lock.json` is regenerated by `npm install`. Expected characteristics:

- `lockfileVersion` stays 3 (npm 11 default; unchanged from current).
- New top-level `node_modules/vinext`, `node_modules/@vitejs/plugin-rsc`,
  `node_modules/@vitejs/plugin-react` (7.x → 5.2.0), `node_modules/@mdx-js/rollup`,
  `node_modules/react-server-dom-webpack`, `node_modules/vite-tsconfig-paths` entries, plus
  vinext's own transitive deps (`@vercel/og`, `@unpic/react`, `sharp`, `image-size`,
  `vite-plugin-commonjs`, `web-vitals`, `srvx`, `turbo-stream`, …).
- `node_modules/vite` moves from 8.1.2 to 7.3.6 (deduped; vitest re-points to it).
- `react`/`react-dom` entries move 19.2.4 → 19.2.7.
- The file is large (~285 KB today) and grows; it is committed verbatim as produced.

## Ordering

Single atomic operation — there is no meaningful sub-ordering. One `npm install` call with all
package specs rewrites both files together, then `npm ci` verifies. Steps are enumerated in
`plan.md`.

## Boundary / interface notes

- **Public interface unchanged.** No import paths, no config surface, no script names change.
  A developer running `npm run dev`/`build`/`test` after this ticket gets *identical* behavior
  to before — the packages are present but not yet wired. This is intentional: the toolchain
  flip is T-006-01-02.
- **Downstream consumers** (T-006-01-02) will `import { vinext } from "vinext"` and
  `@vitejs/plugin-rsc` in a new `vite.config.ts`. This ticket guarantees those imports resolve.
