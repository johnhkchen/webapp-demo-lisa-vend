## The one non-obvious decision worth a reviewer's eye

A naive `npm install vinext vite@^7 @vitejs/plugin-rsc` **fails** with ERESOLVE: npm floats
`@vitejs/plugin-react` (a vinext peer) to its 6.x major, whose peer is `vite ^8`, contradicting
`vite@^7`. The fix ‚Äî and the crux of this ticket ‚Äî is **pinning `@vitejs/plugin-react` to the
5.x line (`^5.2.0`)**, which supports vite 7. If a future change lets that float to 6.x, the
clean-install AC breaks. This is documented in `design.md` and the commit message.

Secondary: **react/react-dom had to bump to 19.2.7** (from 19.2.4) to satisfy the RSC peer
floor (`react-server-dom-webpack@19.2.7` peers `react ^19.2.7`). This is the only change that
reaches into pre-existing *runtime* deps; `next@16.2.9` accepts `^19.0.0`, so it is safe, but a
reviewer should note the app now runs on a slightly newer React patch.
