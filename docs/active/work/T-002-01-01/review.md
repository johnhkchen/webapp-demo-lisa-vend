## Key design decisions (rationale in `design.md`)

1. **`Cell = TetrominoType | null`, not `boolean`.** A settled cell must remember its color for
   rendering and line-fullness checks; a nullable id captures "occupied?" and "which?" in one
   value. `boolean` would force a throwaway parallel color grid later.
2. **Row-major `board[y][x]`.** Matches `emptyBoard(w,h)` shape and the row-wise iteration
   line-clear/collision will need. Intentionally diverges from the renderer's flat array,
   which flattens at its own boundary.
3. **`Piece` stores identity + rotation + position, not resolved cells.** The concrete cell
   offsets per `(type, rotation)` are **T-002-01-02's** shape data; keeping `Piece`
   normalized avoids denormalized state and respects the sibling-ticket boundary.
4. **Types split into `types.ts`, behavior in `board.ts`.** Lets shape (T-002-01-02) and RNG
   (T-002-01-03) import `@/lib/types` without pulling runtime, and reduces same-file
   contention on the shared branch.

## Test coverage

- **Covered:** `emptyBoard` тАФ dimensions for square (10├Ч20) and non-square (3├Ч5) boards,
  all-`null` initial state, and row non-aliasing (the `fill`-trap regression guard).
- **Not directly tested (by design):** the type declarations themselves carry no runtime
  behavior тАФ they are verified transitively by `strict` type-checking (`board.ts` and the
  test import them and must compile) and will be exercised by the tickets that consume them
  (shape data, RNG, collision). Writing a "test" that only re-states a type alias would assert
  nothing the compiler doesn't already enforce.
- **No DOM/integration test:** pure node logic, no rendering; jsdom would be dead weight.

## Open concerns / notes for the reviewer

1. **`RotationState` and `TetrominoType` are defined here but their *data* is not.** This is
   intentional boundary-keeping: T-002-01-02 supplies the shape/offset tables keyed by these
   types, T-002-01-03 the 7-bag. If a reviewer expects piece shapes in this ticket, they are
   correctly absent тАФ that is the next ticket.
2. **`emptyBoard` has no dimension validation** (e.g. negative/zero/non-integer `w`/`h`).
   Deliberately omitted: callers pass the trusted `COLS`/`ROWS` constants, and the engine has
   no untrusted-input surface (no backend, per E-002 non-goals). Add guards only if a future
   caller can supply arbitrary dimensions.
3. **`npm audit` reported advisories** from the transitive dev-dependency tree pulled in by
   Vitest (dev-only, not shipped). Not addressed here тАФ out of scope for this ticket and not a
   production-bundle concern; flag for a maintenance pass if desired.
4. **Committed on shared `main`** per the RDSPI concurrency model; only this ticket's code and
   artifacts were staged, frontmatter and sibling files left for Lisa.
