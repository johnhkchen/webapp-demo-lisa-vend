## Why this was the right-sized change

The scaffold ticket (T-001-01-01) already made Tailwind *apply* — used utilities reached the
browser. A verify-only pass would have re-confirmed the scaffold's incidental work and left the
one real gap (a declared-but-dead `@theme` bridge) in place for the theme epic to trip over.
Expanding the theme (brand palette, glow, motion, fonts) would have stolen that epic's scope.
The chosen delta — consume the *existing* bridge, add no new tokens — is the smallest change
that makes the ticket title honest and hands the theme epic a working pattern. See `design.md`
Decisions 1–2 for the full rationale.

## Open concerns / notes for the reviewer

1. **Rendered pixels are unchanged — that is intended.** The value delivered is *mechanism*
   (dead bridge → proven bridge), not appearance. A reviewer diffing screenshots will see no
   change; the proof is in the emitted CSS, not the pixels.
2. **`@theme inline` semantics.** `inline` means the utilities resolve to `var(--background)` at
   the use site. This is correct for tokens the theme epic will later reassign — changing
   `:root` (or theming via a class/`data-` attribute) will flow through the utilities
   automatically. The theme epic should extend the `:root` palette + `@theme` map, not re-plumb.
3. **Two colors only.** Only `background`/`foreground` are bridged today — that is the full set
   the scaffold defined. Expanding the palette (neon accents, glass surfaces) is explicitly the
   theme epic's job; this ticket intentionally does not pre-empt those names.
4. **No `tailwind.config`.** v4 CSS-first config is the assented fork; none is needed or added.
   A reviewer expecting a JS config file should note this is by design (see `design.md`).
5. **Font stack left as raw CSS.** Tailwind font utilities would require a `--font-*` theme
   token; introducing one is theme-epic scope, so the system-font stack stays in the `body` rule.

## Not in scope (downstream)

Neon/glass theme tokens, glow, animation, self-hosted display font, and any palette expansion —
all reserved for the theme epic, which now inherits a proven, extensible token bridge as settled
ground.

## Bottom line

Small, surgical, verified. The Tailwind fork is now wired end-to-end (utilities **and** the
`@theme` token bridge), the page is unregressed, and the theme epic starts from settled styling
ground instead of a dead declaration. No open blockers.
